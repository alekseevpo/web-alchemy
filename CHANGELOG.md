## [Не опубликовано]

### Добавлено
- Переключатель темы (светлая/тёмная) с использованием next-themes
- Компонент ThemeToggle для переключения между светлой и тёмной темами (расположен в правом верхнем углу header)
- Компонент ThemeProvider для управления темами через next-themes
- Footer с информацией об авторских правах, датой публикации (27 января 2025) и ссылками на нормативные акты:
  - Закон об интеллектуальной собственности Испании
  - Бернская конвенция об охране литературных и художественных произведений
  - Договор ВОИС по авторскому праву (WCT)
  - Договор ВОИС по исполнениям и фонограммам (WPPT)
  - Директива ЕС 2001/29/EC об авторском праве в информационном обществе
- Журнал разработки (CHANGELOG.md) для отслеживания изменений проекта
- Поддержка светлой темы по умолчанию (темная тема доступна через переключатель)
- **Индивидуальные секции комментариев для каждого раздела** - каждый раздел технического задания теперь имеет свою собственную секцию комментариев через Giscus
- **Отдельные обсуждения для каждого раздела** - каждый раздел создает отдельное обсуждение в GitHub Discussions с уникальным идентификатором, что позволяет заказчику комментировать и предлагать изменения по каждому пункту отдельно

### Изменено
- Обновлён layout.tsx для поддержки переключения темы через ThemeProvider
- Добавлен suppressHydrationWarning в html тег для корректной работы next-themes
- Улучшена структура header с позиционированием переключателя темы
- Улучшена структура компонентов для лучшей организации кода
- **Обновлён компонент SectionComment** - теперь каждый раздел имеет полноценную секцию комментариев с заголовком и описанием
- **Улучшена система комментариев** - комментарии теперь доступны непосредственно под каждым разделом, что упрощает процесс обратной связи от заказчика

### Технические детали
- Добавлена зависимость next-themes для управления темами
- Реализовано переключение между светлой и тёмной темами с сохранением выбора пользователя
- Каждая секция комментариев использует mapping="specific" с уникальным data-term для создания отдельных обсуждений в GitHub Discussions

## Формат записей

### Типы изменений
- **Добавлено** - для новых функций
- **Изменено** - для изменений в существующей функциональности
- **Устарело** - для функций, которые скоро будут удалены
- **Удалено** - для удалённых функций
- **Исправлено** - для исправления ошибок
- **Безопасность** - в случае уязвимостей

### Версионирование
- Формат: [MAJOR.MINOR.PATCH] - YYYY-MM-DD
- MAJOR - несовместимые изменения API
- MINOR - новая функциональность с обратной совместимостью
- PATCH - исправления ошибок с обратной совместимостью
